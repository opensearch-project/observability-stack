## 1. Do I need to modify my LLM library calls to get traces?

No. `register()` auto-discovers installed instrumentor packages and activates them. Install the extra for your provider and its calls are traced with no code changes:

```bash
pip install "opensearch-genai-observability-sdk-py[openai]"
```

The `observe()` primitive is only for your own application logic — the orchestration code above the raw LLM calls.

## 2. What happens if I remove `@observe`?

Nothing breaks. `observe()` calls your original function and returns its result unchanged. Remove `@observe` and the function is a plain function again. No SDK calls are embedded inside your logic.

## 3. Can I use the SDK without sending data to AWS?

Yes. For self-hosted OpenSearch or any non-AWS endpoint, just point `endpoint` at your local collector or Data Prepper:

```python
register(endpoint="http://my-collector:4318/v1/traces")
```

<Aside type="tip">
No AWS account or credentials needed for self-hosted OpenSearch.
</Aside>

## 4. What is the performance overhead?

Span creation takes microseconds. The `BatchSpanProcessor` (default) exports in the background, so your code is never blocked waiting on network I/O. Use `batch=False` only when debugging to see spans flush immediately.

## 5. How do I trace a dispatcher where the tool name is dynamic?

Use `name_from` to resolve the span name from a runtime argument:

```python
@observe(op=Op.EXECUTE_TOOL, name_from="tool_name")
def execute_tool(self, tool_name: str, arguments: dict) -> dict:
    return self._tools[tool_name](**arguments)
```

Each call creates a span named `execute_tool <actual_tool_name>`.

## 6. How do I get the trace ID to pass to `score()`?

Read it from the active span context:

```python
from opentelemetry import trace

@observe(op=Op.INVOKE_AGENT, name="my_pipeline")
def run(query: str) -> str:
    ctx = trace.get_current_span().get_span_context()
    trace_id = format(ctx.trace_id, "032x")
    result = do_work(query)
    return result
```

## 7. Do scores appear in the same index as traces in OpenSearch?

Yes. Scores are standard OTEL spans (`gen_ai.evaluation.result`) and travel through the same OTLP pipeline. They land in the same index and can be queried alongside traces in OpenSearch Dashboards.

## 8. What's the difference between `evaluate()` and `Experiment`?

**`evaluate()`** is a full runner — it executes your task function against each test case and runs scorer functions automatically. Use it when you want the SDK to orchestrate the evaluation.

**`Experiment`** is for uploading pre-computed results. Use it when you already have evaluation outputs from another framework (RAGAS, DeepEval, pytest) and want to record them as OTel spans.

Both produce the same `test_suite_run` / `test_case` span structure in OpenSearch.

## 9. Can I use `enrich()` outside of `@observe`?

`enrich()` sets attributes on the currently active span (via `trace.get_current_span()`). It works inside any active span context — whether from `@observe`, a `with observe(...)` block, or even a span from another OTel instrumentation library.

## 10. How do I migrate from `opensearch-genai-sdk-py`?

The new package is `opensearch-genai-observability-sdk-py`. The main API change is that the four separate decorators (`@workflow`, `@agent`, `@tool`, `@task`) are replaced by a single `observe()` primitive with an `op` parameter:

```python
# Old
@agent(name="my_agent")
def run(query): ...

# New
@observe(name="my_agent", op=Op.INVOKE_AGENT)
def run(query): ...
```

See the [migration table](/docs/sdks/python/#migration-from-opensearch-genai-sdk-py) for the full mapping.

## 11. Why do I get `403 SignatureDoesNotMatch` from AWS?

Check that:
- The `region` matches the endpoint's region
- Your credentials have `osis:Ingest` permission (or `es:ESHttpPost` for OpenSearch Service)
- You are using `https://` (HTTP), not `grpc://`

<Aside type="caution">
SigV4 + gRPC is not supported. Use `https://` (OTLP HTTP) for all AWS endpoints.
</Aside>

## 12. What `service` value should I use for SigV4?

| Destination | `service` value |
|---|---|
| OpenSearch Ingestion (OSIS) pipeline | `"osis"` (default) |
| OpenSearch Service domain | `"es"` |
